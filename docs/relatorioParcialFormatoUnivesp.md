RESUMO

Este relatório apresenta o desenvolvimento parcial do sistema “Dentista Organizado”, uma aplicação web voltada à gestão de consultórios odontológicos. O objetivo do projeto é centralizar informações clínicas, otimizar processos administrativos e melhorar a organização de atendimentos por meio de uma plataforma digital integrada. A metodologia adotada baseou-se em princípios de design thinking, com etapas de empatia, definição, ideação, prototipação e testes. Como resultados preliminares, foram implementadas funcionalidades essenciais como autenticação de usuários, gerenciamento de pacientes, controle de agenda, cadastro de convênios e tratamentos, além de um dashboard inicial com indicadores gerenciais. O sistema foi desenvolvido com arquitetura moderna baseada em frontend Angular, backend serverless e banco de dados PostgreSQL. Também foram implementados testes automatizados para garantir qualidade e confiabilidade. Os resultados demonstram a viabilidade da solução proposta, com potencial de evolução para integração completa entre camadas, ampliação de testes e adequação a requisitos de segurança e proteção de dados.

Palavras-chave: sistemas de informação; gestão em saúde; odontologia; desenvolvimento web; arquitetura de software.

1 INTRODUÇÃO

A gestão de consultórios odontológicos envolve múltiplos processos administrativos e clínicos, incluindo controle de pacientes, agendamento de consultas, gerenciamento de tratamentos e acompanhamento financeiro. Em muitos casos, essas atividades são realizadas de forma fragmentada, com uso de ferramentas não integradas, o que gera retrabalho, inconsistência de dados e baixa eficiência operacional.

Diante desse cenário, este projeto propõe o desenvolvimento do sistema “Dentista Organizado”, uma aplicação web que visa centralizar informações e automatizar fluxos de trabalho em clínicas odontológicas. O sistema busca oferecer uma solução integrada que permita o controle de usuários, pacientes, consultas, convênios e procedimentos, além de fornecer indicadores gerenciais para apoio à tomada de decisão.

A escolha do tema justifica-se pela relevância prática no contexto da saúde, bem como pela oportunidade de aplicar conceitos de engenharia de software, arquitetura de sistemas e desenvolvimento web moderno. O projeto está inserido no contexto acadêmico como parte da disciplina de Projeto Integrador I, tendo como objetivo principal a construção de uma solução funcional baseada em problemas reais.

2 DESENVOLVIMENTO

2.1 Objetivos

Objetivo geral:

Desenvolver um sistema web para gestão odontológica capaz de centralizar informações clínicas e administrativas, otimizando o fluxo de trabalho em consultórios.

Objetivos específicos:

* Implementar autenticação e controle de acesso por perfil de usuário;
* Desenvolver funcionalidades de cadastro e gerenciamento de pacientes;
* Criar sistema de agendamento com visualização de agenda;
* Implementar controle de convênios e tratamentos;
* Desenvolver dashboard com indicadores gerenciais;
* Garantir qualidade do sistema por meio de testes automatizados;
* Estruturar arquitetura escalável com separação de camadas.

2.2 Justificativa e delimitação do problema

A gestão de informações em clínicas odontológicas frequentemente apresenta desafios relacionados à fragmentação de dados, ausência de padronização e retrabalho em processos operacionais. Esses problemas impactam diretamente a eficiência do atendimento e a qualidade do serviço prestado.

Dessa forma, o problema que orienta este projeto pode ser definido como: “Como desenvolver um sistema que centralize e organize as informações de um consultório odontológico, reduzindo retrabalho e melhorando a eficiência operacional?”

A proposta justifica-se pela relevância social e prática, ao contribuir para a melhoria da gestão em saúde, além de possibilitar ganhos de produtividade e organização para profissionais da área odontológica.

2.3 Fundamentação teórica

Os sistemas de informação desempenham papel fundamental na gestão de organizações, permitindo a coleta, processamento e análise de dados para suporte à tomada de decisão. No contexto da saúde, esses sistemas contribuem para a melhoria da qualidade do atendimento e para a organização de informações clínicas.

Segundo Laudon e Laudon (2014), sistemas de informação integrados são essenciais para garantir consistência e eficiência nos processos organizacionais. Na área da saúde, a informatização de prontuários e agendamentos tem sido amplamente adotada como forma de reduzir erros e otimizar o atendimento.

Além disso, a arquitetura de software baseada em separação de camadas permite maior manutenção e escalabilidade dos sistemas. Pressman (2016) destaca que a modularização e o desacoplamento são princípios fundamentais para o desenvolvimento de software de qualidade.

O uso de tecnologias web modernas, como aplicações frontend baseadas em frameworks e backend serverless, representa uma tendência atual no desenvolvimento de sistemas, permitindo maior flexibilidade, escalabilidade e redução de custos operacionais.

2.4 Metodologia

A metodologia adotada baseou-se nos princípios de design thinking, estruturada em cinco etapas principais:

Empatia: levantamento de necessidades do contexto de uso, considerando problemas comuns na gestão odontológica.

Definição: identificação dos principais desafios, como fragmentação de dados e falta de integração.

Ideação: definição das funcionalidades e arquitetura do sistema.

Prototipação: desenvolvimento das interfaces utilizando Angular e componentes reutilizáveis.

Testes: validação das funcionalidades, ajustes de usabilidade e estabilização de testes automatizados.

Além disso, foi adotada uma abordagem iterativa de desenvolvimento, com ciclos contínuos de melhoria.

2.5 Resultados preliminares

Como resultado desta etapa, foi desenvolvido um sistema funcional com as seguintes características:

* Implementação de autenticação de usuários;
* Desenvolvimento de páginas para gestão de pacientes, consultas, convênios e tratamentos;
* Criação de agenda com visualização semanal e mensal;
* Implementação de dashboard com indicadores iniciais;
* Estruturação de backend com funções serverless;
* Banco de dados PostgreSQL com tabelas normalizadas;
* Implementação de testes automatizados (205 testes);
* Deploy em ambiente de produção utilizando Vercel.

O sistema apresenta arquitetura baseada em frontend Angular, backend serverless e banco de dados relacional, com separação de responsabilidades e foco em reutilização de componentes.

Os resultados demonstram a viabilidade da solução proposta, bem como sua capacidade de evolução para futuras etapas do projeto.
